# Phase 2 - Data Understanding

## Overview

The **Data Understanding phase** focuses on acquiring, exploring, and validating data required for the ML initiative.

Its purpose is to:
- identify and consolidate relevant data sources  
- understand data structure and key characteristics  
- explore data to generate initial insights  
- assess data quality and identify potential issues  

In this framework, Phase 2 acts as a **data validation and understanding layer**, ensuring that the dataset is suitable for дальнейшей подготовки и моделирования.

---

## Key Outcomes

By the end of this phase, the project must have:

- Identified and consolidated **relevant data sources**  
- Documented **data structure and key attributes**  
- Generated **exploratory insights** to guide further work  
- Assessed **data quality issues and limitations**  
- Established **data readiness for preparation phase**  

This ensures that subsequent data preparation and modeling activities are based on a **well-understood and validated dataset**.

## Deliverables in This Phase

### Technical Artifacts (Level T)

These documents provide structured understanding of data and support reproducibility.

- **P02_Data_Inventory_Report_T**  
  Lists and describes available data sources and their origin.

- **P02_Data_Dictionary_Report_T**  
  Defines structure, attributes, and semantics of the data.

- **P02_Exploratory_Data_Analysis_Report_T**  
  Summarizes key patterns, distributions, and initial insights.

- **P02_Data_Quality_Report_T**  
  Assesses data completeness, consistency, and quality issues.

---

## How This Phase Works

This phase establishes a structured approach to understanding data by:

- consolidating data sources into a unified view  
- documenting data structure and semantics  
- exploring data to uncover patterns and anomalies  
- explicitly identifying data quality risks  

Each artifact contributes to a **transparent and reproducible understanding of the dataset**, ensuring that decisions in subsequent phases are grounded in data reality.

---

## Phase Gate

The phase is completed only after both conditions are met:

### 1. Outcome Validation

All key outcomes of the phase are achieved (see **Key Outcomes** section).

### 2. Data Readiness Validation

- Data sources are consolidated and accessible  
- Data structure is documented and understood  
- Key data quality issues are:
  - either resolved  
  - or explicitly documented and accepted  

A formal decision is made to proceed to:  
**Phase 3 - Data Preparation**
